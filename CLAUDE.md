# 项目开发规范

本文件是项目开发的强制规范，所有后续开发必须严格遵守。

## 全局系统指令（所有项目生效）
### 股票数据开发铁律
1. pysnowball 雪球接口原始数据，**绝对禁止任何单位转换**
2. 禁止修改任何数值，不乘除、不缩放、不四舍五入
3. 接口返回什么就存什么，禁止幻觉、禁止编造计算逻辑
4. 只做数据提取、字段筛选、保存，不做数据加工
5. 如果你改动了代码的逻辑, 那么你就需要更新一下文档, 特别是在git提交代码的时候


## 项目概述
本项目是基于`pysnowball`开发的A股数据获取工具，用于从雪球平台获取股票、基金、指数等多维数据，内置完善的防封策略。
⚠️ 【项目范围强制限定】本项目允许包含**数据下载、数据存储、数据质量校验、定时调度服务**四类功能，禁止开发任何量化回测、选股策略、交易信号、可视化展示、数据分析等超出限定范畴的功能。

## 技术栈要求
- Python 3.10+
- 核心依赖：
  - pysnowball >= 1.2.9（雪球API核心库）
  - pandas >= 2.2.0（数据处理）
  - click >= 8.1.0（命令行框架）
  - python-dotenv >= 1.0.0（环境变量管理）
  - tenacity >= 8.2.0（重试机制）
  - ratelimit >= 2.2.1（请求频率限制）
  - openpyxl >= 3.1.0（Excel文件读写）
  - pandas_market_calendars >= 4.3.0（交易日历支持）
  - exchange_calendars >= 4.2.0（交易所日历支持）
- 开发依赖：
  - pytest >= 7.4.0（单元测试框架）
  - ruff >= 0.3.0（代码规范检查）
- 代码规范：PEP8，使用ruff进行检查

## 开发规范

### 1. 客户端开发规范（client.py）
- 所有API请求必须经过client封装，不得直接调用pysnowball接口
- 防封策略不得修改：1~2秒随机延迟、每分钟30次请求限制、2次重试机制
- 统一错误处理，识别反爬拦截，遇到封禁自动停止所有请求
- 使用单例模式，全局复用同一个客户端实例

### 2. 功能模块开发规范
- 每个功能独立成一个文件，不得混合多个功能到同一个文件
- ⚠️ 强制数据范围要求：支持上证（60开头）、深证主板（00开头）、创业板（30开头）股票，默认过滤ST股票、科创板、北交所（8、4开头）数据，如需要其他类型需单独配置
- 统一的文件保存方法，支持CSV、Excel、JSON三种格式

### 3. 命令行界面规范（main.py）
- 所有提示、帮助文档必须使用中文
- 命令设计简洁易懂，符合用户使用习惯
- 错误提示明确，告知用户问题原因和解决方案
- 支持控制台输出和文件导出两种方式
- ⚠️ 强制要求：main.py仅作为统一功能入口，只做命令定义和参数解析，不编写具体功能实现代码，所有功能逻辑必须放到对应模块文件中
- main.py中的每个命令必须添加清晰的中文注释说明功能 
- main.py同级不允许出现别的py文件

### 4. 代码规范
- 必须添加中文注释，关键逻辑注释清晰
- 函数、变量命名见名知意，禁止使用无意义缩写
- 代码必须通过ruff检查，无语法错误和规范问题
- 新增功能必须配套测试用例

## 安全规范
- 禁止使用本工具进行大规模爬虫，合理控制请求频率
- 禁止将获取的数据用于商业用途，遵守雪球平台协议
- 遇到反爬拦截时立即停止请求，不得绕过限制

## 代码提交规范 执行【Git 提交前代码审查】，仅审查本次新增/修改的代码。
请以多Agent交叉审查模式检查：

1. 合规检查：是否违反项目 claude.md 中的所有规则
2. 代码规范：语法、风格、可读性、注释是否合理
3. 逻辑检查：是否有明显bug、未处理异常、无效逻辑
4. 安全检查：无硬编码、无敏感信息泄露
5. 冗余检查：无无用代码、无调试代码、无幻觉代码
