# 项目开发规范

本文件是项目开发的强制规范，所有后续开发必须严格遵守。

## 项目概述
本项目是基于`pysnowball`开发的A股数据获取工具，用于从雪球平台获取股票、基金、指数等多维数据，内置完善的防封策略。

## 技术栈要求
- Python 3.10+
- 核心依赖：pysnowball >= 1.2.9
- 其他依赖：pandas, click, python-dotenv, tenacity, ratelimit, openpyxl
- 代码规范：PEP8，使用ruff进行检查

## 开发规范

### 1. 客户端开发规范（client.py）
- 所有API请求必须经过client封装，不得直接调用pysnowball接口
- 防封策略不得修改：1~2秒随机延迟、每分钟30次请求限制、2次重试机制
- 统一错误处理，识别反爬拦截，遇到封禁自动停止所有请求
- 使用单例模式，全局复用同一个客户端实例

### 2. 功能模块开发规范
- 每个功能独立成一个文件，不得混合多个功能到同一个文件
- ⚠️ 强制数据范围要求：仅支持上证（60开头）、深证主板（00开头）股票，默认过滤ST股票、创业板（30开头）、北交所（8、4开头）数据，如需要其他类型需单独配置
- 所有字段名使用中文，方便用户使用
- 数值类字段需要做单位转换（比如元转万元、百分比转换）
- 统一的文件保存方法，支持CSV、Excel、JSON三种格式

### 3. 命令行界面规范（cli.py）
- 所有提示、帮助文档必须使用中文
- 命令设计简洁易懂，符合用户使用习惯
- 错误提示明确，告知用户问题原因和解决方案
- 支持控制台输出和文件导出两种方式
- ⚠️ 强制要求：cli.py仅作为统一功能入口，只做命令定义和参数解析，不编写具体功能实现代码，所有功能逻辑必须放到对应模块文件中
- cli.py中的每个命令必须添加清晰的中文注释说明功能

### 4. 代码规范
- 必须添加中文注释，关键逻辑注释清晰
- 函数、变量命名见名知意，禁止使用无意义缩写
- 代码必须通过ruff检查，无语法错误和规范问题
- 新增功能必须配套测试用例

## 安全规范
- 禁止将包含个人Cookie的.env文件提交到Git
- 禁止使用本工具进行大规模爬虫，合理控制请求频率
- 禁止将获取的数据用于商业用途，遵守雪球平台协议
- 遇到反爬拦截时立即停止请求，不得绕过限制

## 版本管理规范
- 版本号格式：主版本号.次版本号.修订号
- 新增功能升级次版本号，修复bug升级修订号
- 每个版本更新必须记录更新内容到CHANGELOG.md（如需）
